Activity Name Type Date Number of hours claimed: How did it support my development? What happened and what did I take away from it? What are my next steps?
Filter contents Course 18/01/2025 1.25 As a cyber security professional is important that I develop an understanding of Linux which is used as the base for Android, Web servers, embedded devices etc making it a powerhouse in the IT world which everything runs on. I read and practiced tools like grep, cut, wc, awk, more, less, head, tail, sort, tr, column, and sed. I learnt the tools as effective methods for filtering program outputs from cURL, wget, and cat /etc/passwd Move on to “Regular Expressions” or otherwise known as regex
RegEx Course 20/01/2025 0.25 Most modern programming and markup languages support RegEx. It is used to pattern match data, useful with stdout. As a cyber professional I will need to be able to search through logs and find matches like who accessed what. I practiced using RegEx with the grep command on /etc/passwd I learnt pattern matching in the terminal which is a powerful tool for filtering out information or programs for other steps Move onto permission management
Permission Management Course 20/01/2025 0.08 As cyber professional it is important to ensure access is only granted where necessary to protect information and curb insider threats. I learnt about octal notation to set permissions. Example includes 754 and permissions include read(r), write(w) and execute(x) with first part being applied to the owner, next to the group and last applied to others. The numbers come from binary notation where a 1 is activate permission at this location. See mapping here with 7/5/4 being 111/101/100 --> rwx/r_x/r__ Setting permissions and checking they are correctly set is no easy task with the octal notation at first due to the brain being unfamiliar with the patterns. With practice, it will become second nature and handy tool. Move onto user management. Practice setting permissions to rewire the brain to understanding the patterns.
User, Package, Service and Process Management Course 21/01/2025 1 A cyber security professional needs to know how to manage users, install and manage their tools effectively. I learnt user management with useradd, userdel, etc. I learnt the different package managers including apt, dpkg, git, gem, pip, aptitude, and snap. I also learnt how to push stop, kill, or push processes into the background or foreground. It meant I could start seeing how we can use various tools and at the same time to reach a desired effect whether that is defensive or offensive. Continue with training
Task scheduling Course 29/01/2025 0.5 Automating scripts and tasks in Linux is essential for ensuring systems stay updated and secure, this is the responsibility of a System Operations Centre Analyst I learnt task scheduling using Cron and Systemd. Systemd needs timers and is linked to another process potentially system process while Cron runs task based on the daemon time. It meant I can now carry out a key responsibility of being a Cyber Security professional. Continue with training

Backup and Restore Course 1/02/2025 0.5 Backing up and Restoring information securely is important in a system administrators’ role. Working with Web Services Course 2/02/2025 0.75 Understanding web servers is important as they host consumer facing content which is most prone to attacks and thus need protecting by us professionals I learnt Php, Python3, Npm and Apache2 servers as well as using Curl and Wget to assess them.
